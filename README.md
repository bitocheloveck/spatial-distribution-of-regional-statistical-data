# Пространственное распределение региональных статистических данных по территории субъекта РФ на сетке 1×1 км без метода усреднения

## Проект подготовлен Ивашко Даниилом и Носыриным Владиславом

**Основная цель**: создания модели, распределяющей когорту демографических региональных данных по сетке 1х1 км 

---

### **1. Сбор данных**

В качестве основы распределения показателя были выбраны данные, характеризующие географическое пространство, а именно:

* Рельеф территории (Источник: [FABDEM V1-2: Глобальная карта высот с разрешением 30 м, без учета лесов и зданий](https://data.bris.ac.uk/data/dataset/s5hqmjcdj8yo2ibzi9b4ew3sn));
* Ночные огни (Источник: [Безоблачная композиция в дневном и ночном диапазоне системы видимого и инфракрасного излучения (VIIRS)](https://eogdata.mines.edu/products/vnl/#:~:text=Monthly%20Cloud%2Dfree%20DNB%20Composite));
* Застройка территории (Источник: OSM, тег: `buildings`, вид данных: полигоны, инструмент получения данных: плагин QGIS);
* Транспортная сеть (Источник: OSM, тег: `highway`, вид данных: Линии или дуги, инструмент получения данных: плагин QGIS); 
* Природные площади (Леса, поля и т.д.)(Источник: OSM, тег: `natural`, вид данных: полигоны, инструмент получения данных: плагин QGIS);
* Административные границы (Источник: OSM, тег: `place`, вид данных: полигоны, инструмент получения данных: плагин QGIS).

Также для построения бейзлайна модели были собраны данные о численности населения по муниципальным образованиям (Источник: [Росстат: 
Численность населения Российской Федерации по муниципальным образованиям на 1 января 2024 года](https://rosstat.gov.ru/folder/11110/document/13282)).

### **2. Формирование сетки 1х1 км и прокси-бейзлайна**

#### Теоритическая основа создания прокси-бейзлайна

Для построения базового распределения демографических показателей был выбран следующий подход:
1. Демографический показатель может быть распределен только в местах проживания населения и вблизи них.
2. Население распределено неравномерно в рамках районов своего проживания. Плотность населения отличается.

Исходя из этой основы данные были подготовлены следующим образом:
1. Численность населения была взята на уровень ниже регионального - по муниципальным округам.
2. Внутри муниципальных округов население было распределено по стеке 1х1 пропорционально плотности застройки внутри клетки (Доли территории клетки, занятой зданиями)

#### Ключевые допущения

1. Пропорциональность застройки и населения
    * *Допущение*: Численность населения в клетке пропорциональна площади застройки.
    * *Ограничение*: Не учитывает этажность зданий, тип застройки (многоквартирная/частная) и функциональное назначение (офисы, склады). Это может приводить к занижению плотности в центральных районах и завышению - в промзонах.
2. Однородность внутри муниципалитета
    * *Допущение*: Отношение «население / площадь застройки» считается константой в границах одного муниципального образования.
    * *Ограничение*: В крупных городских округах плотность может различаться на порядок между центром и периферией. Модель сглаживает эти различия.
3. Полнота и актуальность исходных данных
    * *Допущение*: Данные OSM достаточно полны для отражения реальной застройки.
    * *Ограничение*: В сельских и удалённых районах покрытие OSM может быть фрагментарным, что приводит к «пропуску» реально существующего населения.
4. Стационарность ночных огней
    * *Допущение*: Интенсивность ночного излучения стабильно коррелирует с плотностью заселения.
    * *Ограничение*: Промышленные объекты, транспортные развязки и туристические зоны могут давать высокий сигнал при нулевом постоянном населении.

### 3. Итоговая предобработка данных для модели

#### Бейзлайн

В качестве обязательного baseline используется равномерное распределение населения внутри муниципалитета.

Для каждой ячейки муниципалитета считается одинаковая доля:
uniform_share = 1 / number_of_cells_in_municipality

Затем население распределяется так:
uniform_population = uniform_share × total_population

Этот baseline нужен, чтобы показать, что предложенный метод лучше простого равномерного разнесения показателя по территории.

#### Proxy-based распределение

Изначальный прокси-бейзлайн плохо апроксимировался моделью, поэтому была создана иная целевая переменная — `proxy_share`.

Для каждой ячейки рассчитывается `proxy_score`, отражающий относительную "привлекательность" территории для размещения населения:
proxy_score = 0.35 × log(total_building_area)
+ 0.25 × log(number_of_buildings)
+ 0.20 × log(road_length)
+ 0.20 × log(ntl_mean)
- 0.30 × nature_density

**Логика признаков:**

- `total_building_area` — чем больше площадь зданий, тем выше вероятность населения;
- `number_of_buildings` — учитывает плотность застройки;
- `road_length` — отражает инфраструктурную доступность;
- `ntl_mean` — ночные огни используются как индикатор человеческой активности;
- `nature_density` — природные территории снижают вероятность размещения населения.

Использованные пространственные признаки соответствуют подходам, применяемым в проектах gridded population mapping, таких как WorldPop. Весовые коэффициенты были заданы эвристически на основе предполагаемой значимости признаков для пространственной концентрации населения.

Пустым ячейкам задаётся очень низкий score, чтобы они почти не получали население.

Затем `proxy_score` преобразуется в долю населения внутри каждого муниципалитета через softmax:
proxy_share = softmax(proxy_score внутри municipality)

#### Смешанное proxy-распределение

Чтобы proxy-распределение не было слишком жёстким, оно смешивается с равномерным baseline:
mixed_share = 0.85 × proxy_share + 0.15 × uniform_share

После этого `mixed_share` нормализуется внутри каждого муниципалитета, чтобы сумма долей была равна 1.

`Mixed_share` используется как обучающая целевая переменная для regression-модели.

#### Feature engineering

Для обучения моделей используются исходные пространственные признаки и дополнительные engineered features.

**Добавлены следующие логарифмированные признаки:**

- `log_building_area`
- `log_number_of_buildings`
- `log_road_length`
- `log_nature_area`
- `log_ntl_mean`
- `log_ntl_sum`
- `log_ntl_max`

Они уменьшают влияние экстремальных значений и делают распределения более стабильными.

**Также добавлены interaction-признаки:**

built_road_interaction = building_coverage × road_density
built_ntl_interaction = building_coverage × ntl_mean
road_ntl_interaction = road_density × ntl_mean

Они отражают совместный эффект застройки, дорог и ночного освещения.

**Дополнительно рассчитаны:**

- `building_per_road`
- `urban_nature_ratio`
- `nature_penalty`
- `terrain_roughness`

Эти признаки помогают учитывать структуру территории, природные ограничения и особенности рельефа.

### 4. Обучение моделей

#### Regression-модель

Первая модель — **CatBoostRegressor**.

Она обучается предсказывать `log(mixed_share)`.

Логарифмирование используется потому, что доли населения сильно скошены: большинство ячеек получает очень маленькие значения, а небольшое число ячеек концентрирует значимую часть населения.

После предсказания результат переводится обратно:

pred_score = exp(pred_log_share_raw)

Затем значения нормализуются внутри каждого муниципалитета:

pred_share = pred_score / sum(pred_score внутри municipality)

Это гарантирует **conservation constraint**:

sum(pred_share внутри municipality) = 1

#### Ranking-модель

Вторая модель — **CatBoostRanker**.

Она решает не задачу точного предсказания доли, а задачу ранжирования ячеек внутри муниципалитета.

Целевая переменная:

relevance = log1p(proxy_share / mean(proxy_share))

Ranking-модель учится определять, какие ячейки внутри муниципалитета являются более важными для распределения населения.

После получения `rank_score_raw` значения преобразуются в доли через softmax внутри муниципалитета:

rank_share = softmax(rank_score_raw внутри municipality)

#### Гибридная модель

Итоговая модель объединяет regression и ranking подходы:

hybrid_share = beta × pred_share + (1 - beta) × rank_share

В проекте используется:

beta = 0.7

Это означает, что итоговая модель больше опирается на regression-модель, так как она лучше восстанавливает абсолютные значения долей, но при этом учитывает ranking-модель, которая хорошо сохраняет относительную пространственную структуру.

После смешивания `hybrid_share` снова нормализуется внутри муниципалитета:

sum(hybrid_share внутри municipality) = 1

И итоговое население ячейки считается так:

hybrid_population = hybrid_share × total_population

---

### 5. Итоговая модель

В качестве основной итоговой модели выбрана **гибридная модель Regression + Ranking**.

**Причины выбора:**

- regression-модель лучше оценивает абсолютные доли населения;
- ranking-модель лучше отражает относительную важность ячеек;
- параметр beta позволяет регулировать баланс между количественной точностью и пространственной структурой;
- итоговое распределение сохраняет сумму населения внутри каждого муниципалитета;
- метод воспроизводим и может быть применен к соседним и похожим регионам, например к Курской области.
